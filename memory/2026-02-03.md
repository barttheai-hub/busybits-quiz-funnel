## Kling 2.6 API notes (from docs)
- API domain: `https://api-singapore.klingai.com`.
- Auth: JWT (HS256) signed with SecretKey; payload uses `iss=<AccessKey>`, `exp=now+1800`, `nbf=now-5`; header `Authorization: Bearer <token>`.
- Billing: Kling API uses **prepaid resource packs**; “task point deduction” may produce watermarked outputs.
- Lip-sync is 2-step:
  1) `POST /v1/videos/Identify-face` with `video_id` or `video_url`.
  2) `POST /v1/videos/advanced-lip-sync` with `session_id` + `face_choose` + audio.
     - Audio: `sound_file` supports base64 or accessible URL; formats mp3/wav/m4a ≤5MB.
- To get a video from a photo + prompt in v2.6: use **Image to Video**:
  - `POST /v1/videos/image2video` with `model_name: kling-v2-6`, `image` (URL/base64), `prompt`, `mode`, `duration`, `sound` (keep off since we use ElevenLabs).
  - Intended flow: `image2video` (from avatar still) → `Identify-face` → `advanced-lip-sync`.

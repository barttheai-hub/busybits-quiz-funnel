# BusyBits newsletter — growth + engagement experiments (high leverage)

Goal context (from USER.md): ~95k subs, push toward ~50% open rate, sponsors, growth via paid acquisition.

## 0) Measurement + hygiene (do first)
- Confirm deliverability basics: SPF/DKIM/DMARC, custom tracking domain (if available), and sender reputation.
- Segment list into:
  - Engaged 30d (opened/clicked)
  - Warm 90d
  - Cold 180d+
- Establish 2 baseline metrics per send:
  - Open rate by segment
  - Click-to-open rate (CTOR)

## 1) Open-rate lift (fast tests)
### Experiment A — Subject line system (2-week sprint)
- Write 10 subject lines for each email:
  - Curiosity
  - Benefit
  - Contrarian
  - "Number" style
  - Ultra-short (1–3 words)
- A/B test at least 2 variants to a randomized slice of Engaged 30d; roll winner to rest.

### Experiment B — From-name test
- Test 3 options over 6 sends:
  1) “Ziga”
  2) “Ziga @ BusyBits”
  3) “BusyBits”
Measure: open rate + spam complaints.

### Experiment C — Send-time personalization
- Use “send by time zone” + test 2 send windows:
  - AM commute vs evening wind-down
- If beehiiv supports smart send-time, enable for engaged segment.

## 2) Content format that drives engagement (and sponsors)
### Experiment D — The “One Big Idea” template
Structure:
1) Hook (1–2 lines)
2) The idea (4–7 bullets)
3) 1 proof link
4) 1 action step
5) 1 sponsor slot

### Experiment E — “3 quick wins” issue
- Short, skimmable, highly actionable.
- Track CTOR + reply rate.

## 3) List cleanup / reactivation (protect deliverability)
### Experiment F — Cold segment reactivation ladder
- For Cold 180d+, run a 3-email sequence:
  1) “Still want this?” (one-click keep me)
  2) “Best of the year” (high value)
  3) “I’ll remove you tomorrow” (final)
- Remove unresponsive to protect inbox placement.

## 4) Growth loops (cheap subscribers)
### Experiment G — Referral incentive refresh
- Add a single compelling milestone:
  - 3 referrals → “BusyBits swipe file”
  - 10 referrals → “monthly teardown PDF”
- Put referral CTA mid-email (not only footer).

### Experiment H — Social validation gate
- Borrow from top beehiiv operators: test ideas on socials first.
  - Only promote/email topics that earn ≥15 comments or strong saves.

## 5) Paid acquisition (if CAC works)
### Experiment I — Lead magnet + paid to subscribe
- Build 1 lead magnet that matches sponsor buyer intent (e.g., “Fitness offer teardown pack”).
- Run ads to that magnet + newsletter opt-in.
- Track downstream: engaged opens within 7 days.

## Next steps
1) Pick 3 experiments to run next 14 days.
2) Define success metrics + sample size.
3) Create the exact copy blocks (subject line sets, reactivation emails, referral CTA modules).
